# 화이트 기술 학습 워크플로우 시각화 설계

## 목적

앞 장면의 질문인 “내 커리어에 필요한 건 무엇일까?”에 답하는 독립 Canvas 시각화다. 화면에 흩어진 여러 기술 중 네 가지 기술만 선택되고, 배워야 할 순서인 `Java → Spring → Docker → AWS`로 연결되는 과정을 보여준다.

분야별 분류 장면을 반복하지 않고, 많은 선택지에서 개인에게 필요한 학습 경로가 추려진다는 메시지에 집중한다.

## 범위

- 기존 시각화 파일과 공통 렌더링 코드는 수정하지 않는다.
- 별도의 새 시각화 파일과 전용 테스트 파일을 추가한다.
- 완성 후 시각화 레지스트리에 새 항목만 등록한다.
- 사용자 입력이나 API 연동 없이 고정된 데모 데이터로 재생한다.
- 배경은 기존 다크 스테이지와 구분되는 화이트 스테이지로 구성한다.

## 장면 구성

기준 캔버스는 960×540이며 다른 해상도에서는 기존 시각화와 동일하게 비례 확대·축소한다.

1. **탐색**: 14개의 모노톤 기술 칩이 화면에 무작위처럼 보이는 고정 위치로 흩어져 있다.
2. **선택**: `Java`, `Spring`, `Docker`, `AWS`가 순서대로 어두운 선택 상태로 강조된다.
3. **정렬**: 선택된 네 칩이 중앙의 수평 워크플로우 위치로 부드럽게 이동한다.
4. **연결**: 각 칩 사이에 선과 화살표가 왼쪽에서 오른쪽으로 차례대로 그려진다.
5. **정리**: 선택되지 않은 기술 칩은 투명도가 낮아지며 사라진다.
6. **완성**: 네 칩과 연결선이 남고, 하단에 `나에게 필요한 기술을, 배워야 할 순서대로` 문구가 나타난다.

전체 애니메이션은 7,000ms 동안 재생한 뒤 자연스럽게 반복한다. 마지막 1,200ms에는 완성된 경로를 고정해 충분히 읽을 수 있게 한다.

## 기술 데이터

선택 기술과 순서는 다음과 같이 고정한다.

| 순서 | 기술 | 역할 |
|---:|---|---|
| 1 | Java | 언어 기반 |
| 2 | Spring | 백엔드 프레임워크 |
| 3 | Docker | 실행 환경 패키징 |
| 4 | AWS | 배포·운영 환경 |

배경 기술은 React, Python, MySQL, Git, Kubernetes, TypeScript, Kafka, Redis 등으로 구성한다. 선택되지 않은 기술은 연결선에 참여하지 않는다.

## 시각 스타일

- **배경**: 따뜻한 화이트 `#F7F7F5` 계열
- **배경 칩**: 흰색 채움, 연회색 테두리, 중간 회색 텍스트
- **선택 칩**: 검은색에 가까운 채움, 흰색 텍스트, 절제된 그림자
- **연결선과 화살표**: 진한 차콜, 연결 진행률에 맞춰 왼쪽에서 오른쪽으로 노출
- **보조 요소**: 매우 옅은 회색 그리드 또는 점 패턴으로 빈 흰 배경의 깊이만 보완
- **최종 문구**: 검은색 중심 정렬

색상 강조는 선택 여부만 표현한다. 기술 분야별 색상은 사용하지 않는다.

## 모션 원칙

- 초기 기술 위치는 실행마다 변하지 않는다. 영상 재생과 테스트 결과가 항상 동일해야 한다.
- 선택 기술은 강조된 뒤 중앙으로 이동하며, 이동 중 겹치거나 화면 밖으로 나가지 않는다.
- 연결선은 칩 정렬이 끝난 뒤 생성해 인과 순서를 분명히 한다.
- 배경 기술의 소멸은 갑작스러운 삭제가 아니라 페이드아웃으로 처리한다.
- 최종 문구는 경로 완성 후 페이드인한다.
- 반복 시작점과 끝점에서 플래시나 잔상이 생기지 않도록 프레임 상태를 진행률의 순수 함수로 계산한다.

## 코드 구조

새 파일은 다음 책임을 분리한다.

- 고정 기술 목록과 초기 좌표
- 진행률을 칩 상태, 연결 상태, 문구 투명도로 변환하는 프레임 계산 함수
- 화이트 배경, 칩, 연결선, 문구를 그리는 전용 Canvas 렌더 함수
- 레지스트리에 등록할 `VizDef`

기존 모노 기술 더미 및 이력서 기술 분류 구현은 가져다 수정하지 않는다. 필요한 기하 계산은 새 파일 내부에 두며, 기존 공통 유틸리티 중 부작용 없는 함수만 읽기 전용으로 사용한다.

## 검증 기준

- 시작 프레임에는 모든 기술 칩이 흰 배경에 흩어져 있다.
- 중간 프레임에서 선택 기술이 순서대로 강조되고 중앙으로 이동한다.
- 마지막 프레임에는 `Java → Spring → Docker → AWS`만 연결된 상태로 남는다.
- 선택되지 않은 기술은 마지막 프레임에서 보이지 않는다.
- 최종 문구는 경로 완성 이후에만 보인다.
- 같은 진행률과 해상도는 항상 같은 상태를 반환한다.
- 기존 시각화 테스트와 프로덕션 빌드가 계속 통과한다.
- 로컬 `/ppt-visual-maker`에서 항목 선택, 반복 재생, 배속 변경 시 화면이 깨지지 않는다.
