# 서비스 타이틀 전환 시각화 설계

## 목적

`기술 더미 질문` 다음에 재생되는 독립 Canvas 시각화다. 질문 장면에 남은 모노 기술 더미를 이력서 데이터로 압축하고, 오른쪽에서 들어오는 채용시장 데이터와 중앙에서 결합해 `Career Fit` 서비스 타이틀을 만든다.

텍스트 설명 대신 형태와 이동으로 `내 이력서 × 채용시장 데이터 = 커리어 방향`이라는 서비스 핵심을 전달한다.

## 범위

- 새 시각화 파일과 전용 테스트 파일을 추가한다.
- 기존 `기술 더미 질문`의 렌더링과 재생 시간은 수정하지 않는다.
- 완성 후 시각화 레지스트리에 기능 시각화 항목을 하나 추가한다.
- API나 사용자 입력 없이 고정된 데모 데이터로 재생한다.
- 화면에 표시하는 텍스트는 최종 서비스명과 한 줄 설명으로 제한한다.

## 장면 연결

새 장면의 첫 프레임은 `기술 더미 질문`의 마지막 프레임과 동일해야 한다.

- 배경: `WHITE_MONO_STAGE_COLOR` (`#F7F7F5`)
- 칩 데이터: `getTechChipPileStates(1, width, height)`
- 칩 스타일: `MONO_CHIP_STYLES`
- 바닥선: 기존 두 장면과 동일한 좌표와 색상
- 질문 문구: 표시하지 않음

따라서 두 영상을 순서대로 배치했을 때 배경, 칩 위치, 그림자, 바닥선이 움직이지 않고 바로 다음 모션으로 이어진다.

## 타임라인

전체 재생 시간은 7,000ms다.

### 0–500ms: 연결 유지

- 완성된 기술 더미를 그대로 유지한다.
- 서비스명과 데이터 소스 형태는 보이지 않는다.

### 500–1,900ms: 이력서로 압축

- 16개 기술 칩이 왼쪽으로 이동하며 크기가 작아진다.
- 칩은 한 장의 세로형 이력서 윤곽 안에 정렬된 짧은 줄과 점 형태로 수렴한다.
- 바닥선은 이 구간에서 페이드아웃한다.
- 별도의 `이력서` 텍스트는 표시하지 않는다.

### 1,500–2,800ms: 시장 데이터 등장

- 오른쪽에서 24개의 작은 공고 카드와 데이터 점이 고정된 순서로 나타난다.
- 공고 카드는 짧은 가로선이 들어간 모노톤 사각형으로 표현한다.
- 요소 수와 좌표는 고정해 매 재생 결과가 동일하게 한다.
- `시장`, `공고` 같은 설명 텍스트는 표시하지 않는다.

### 2,600–4,200ms: 두 데이터 흐름 결합

- 왼쪽 이력서와 오른쪽 시장 요소에서 작은 입자들이 중앙으로 이동한다.
- 이동 경로는 서로 다른 두 곡선이 중앙에서 맞물리는 형태다.
- 중앙 도착 순서에 맞춰 서비스 심볼의 두 부분이 조립된다.
- 이력서와 시장 원본 형태는 입자 이동이 진행될수록 흐려진다.

### 4,100–4,900ms: Fit 심볼 완성

- 두 데이터 흐름이 맞물리며 중앙에 36×36px 기준 크기의 라운드 정사각형 심볼을 만든다.
- 심볼은 기존 `Career Fit`의 작은 그라데이션 정사각형을 화이트 배경에 맞게 반전한 형태다.
- 짙은 외곽과 밝은 대각 하이라이트로 기존 브랜드 인상을 유지한다.
- 완성 순간 한 번만 미세하게 확대됐다가 원래 크기로 돌아온다.

### 4,900–7,000ms: 서비스 타이틀

- 심볼 오른쪽에 `Career Fit`이 4,900–5,400ms 동안 페이드인해 하나의 가로 브랜드 행을 만든다.
- 브랜드 행 아래 중앙에 한 줄 설명 `데이터로 찾는 나의 커리어 방향`이 5,200–5,600ms 동안 페이드인한다.
- 마지막 1,400ms에는 심볼, 서비스명, 한 줄 설명을 고정해 읽을 시간을 확보한다.

## 시각 스타일

- 배경: `#F7F7F5`
- 주요 형태와 서비스명: `#18181B`
- 보조 선과 데이터: `#71717A`, `#A1A1AA`, `#D4D4D8`
- 강조색은 사용하지 않고 흑·백·회색만 사용한다.
- 중앙 심볼 이외의 요소에는 강한 빛 번짐을 사용하지 않는다.
- 폰트는 기존 Canvas 시각화의 `FONT` 상수를 사용한다.
- 최종 서비스명은 굵게, 한 줄 설명은 중간 굵기의 회색으로 표시한다.

## 상태 모델

프레임 계산 함수는 진행률과 해상도를 받아 다음 상태를 반환한다.

- 기술 칩별 위치, 크기, 투명도, 이력서 수렴 진행률
- 이력서 윤곽 투명도
- 시장 카드·점별 위치와 투명도
- 좌우 데이터 입자별 곡선 진행률과 투명도
- 중앙 심볼 조립 진행률과 크기
- 서비스명과 한 줄 설명의 투명도

모든 좌표와 순서는 고정 배열로 정의하고 `Math.random()`을 사용하지 않는다.

## 검증 기준

- 첫 프레임의 기술 더미가 `기술 더미 질문` 마지막 프레임과 동일하다.
- 500ms 전에는 이력서, 시장 데이터, 타이틀이 보이지 않는다.
- 이력서와 시장 데이터가 텍스트 라벨 없이 형태만으로 구분된다.
- 좌우 데이터가 중앙으로 이동한 뒤 Fit 심볼이 완성된다.
- 마지막 프레임에는 `Career Fit`과 `데이터로 찾는 나의 커리어 방향`만 중심 요소로 남는다.
- 같은 진행률과 해상도는 항상 같은 상태를 반환한다.
- 960×540, 1920×1080, 4K에서 요소가 화면 밖으로 나가거나 겹치지 않는다.
- 기존 테스트와 프로덕션 빌드가 계속 통과한다.
- 로컬 `/ppt-visual-maker`에서 질문 장면의 끝과 타이틀 장면의 시작 사이에 시각적 점프가 없다.
