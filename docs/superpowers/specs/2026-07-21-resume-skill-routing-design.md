# 이력서 기술 분야 분류 시각화 설계

## 목적

한 장의 이력서에서 사용 가능한 기술 8개가 순서대로 추출되고, 각 기술이 백엔드·프론트·데이터·인프라 분야로 이동해 분류되는 과정을 보여주는 별도 Canvas 시각화를 만든다. 별, 점군, 입자, 별자리 표현은 사용하지 않는다.

## 기술과 분야

| 분야 | 기술 | 칩 명도 |
|---|---|---|
| 백엔드 | Java, Spring | 흰색 |
| 프론트 | React | 밝은 회색 |
| 데이터 | MySQL, Python | 중간 회색 |
| 인프라 | Docker, AWS, Git | 검정 |

검정 칩은 어두운 스테이지에서 구분되도록 밝은 테두리와 흰 글자를 사용한다. 나머지 칩은 배경 명도에 맞춰 어두운 글자와 아이콘을 사용한다.

## 화면 구성

- Career Fit의 `#282828` 기본 스테이지와 Pretendard 타이포그래피를 유지한다.
- 왼쪽 중앙에 실제 개인정보가 없는 모노톤 이력서 한 장을 배치한다.
- 오른쪽에는 `백엔드`, `프론트`, `데이터`, `인프라` 목적지를 위에서 아래로 세로 정렬한다.
- 이력서와 각 목적지 사이에는 분야 명도와 동일한 얇은 곡선 레인을 표시한다.
- 중앙 허브, 방사형 점, 별, 반짝임, 입자 군집은 그리지 않는다.
- 상단 라벨은 `문제 인식 · 이력서 속 기술 분류`, 하단 캡션은 진행 상태에 따라 변경한다.

## 애니메이션

- 전체 재생 시간은 9000ms다.
- 시작 0.5초 동안 이력서와 네 분야 레인이 먼저 나타난다.
- 기술 칩은 `Java, React, Docker, MySQL, Spring, AWS, Python, Git` 순서로 약 0.65초 간격을 두고 이력서 문장 사이에서 하나씩 생성된다.
- 각 칩은 해당 분야의 곡선 레인을 따라 이동하며, 이동 후반에 감속한다.
- 칩은 목적지에 도착하는 마지막 구간에서 크기와 투명도가 함께 줄어들어 자연스럽게 흡수된다.
- 도착 시 해당 분야 라벨과 `분류 완료 n` 카운트가 짧게 밝아졌다가 기본 명도로 돌아간다.
- 마지막 1초에는 모든 칩이 사라지고 이력서, 네 레인, 최종 분야별 카운트만 정지 상태로 남는다.
- 동일 진행률과 해상도는 항상 동일한 프레임을 출력하며 난수를 사용하지 않는다.

## 코드 구조

- 새 파일 `src/ppt/viz/resume-skill-routing.ts`에 기술 데이터, 상태 계산, Canvas 렌더러, 시각화 정의를 둔다.
- 새 테스트 `tests/resumeSkillRouting.test.ts`에서 기술 매핑, 시점별 이동·소멸, 최종 카운트, 메타데이터를 검증한다.
- `package.json`의 테스트 명령에 새 테스트를 추가한다.
- `src/ppt/vizRegistry.ts`에는 새 시각화 import와 항목만 추가한다.
- 시각화 ID는 `resume-skill-routing`, 제목은 `이력서 기술 분류`, 카테고리는 `feature`, 재생 시간은 9000ms로 정의한다.
- 기존 `problem.ts`, `resume-stack-clean.ts`, `tech-chip-pile.ts`, `tech-chip-pile-mono.ts`, `common.ts`는 수정하지 않는다.

## 상태 계산

각 기술 상태는 이름, 분야, 명도, 표시 여부, 이동 여부, 도착 여부, 위치, 크기, 투명도를 포함한다. 시작점은 이력서 내부의 고정 문장 행이며 도착점은 해당 분야 라벨 앞의 고정 좌표다. 위치는 2차 베지어 곡선과 ease-in-out 보간으로 계산한다.

분야별 완료 카운트는 현재 시간까지 도착한 기술 수에서 계산한다. 별도 가변 상태를 사용하지 않아 탐색, 반복 재생, MP4 프레임 렌더 결과가 일치하게 한다.

## 검증

- 기술 8개가 지정된 네 분야에 정확히 매핑되는지 테스트한다.
- 시작 프레임에는 이동 칩이 없고, 중간 프레임에는 일부 칩이 이동하거나 도착했으며, 마지막 1초에는 모든 칩이 보이지 않는지 테스트한다.
- 최종 분야별 완료 수가 백엔드 2, 프론트 1, 데이터 2, 인프라 3인지 테스트한다.
- 모노톤 4단계 외의 카테고리 색상이 사용되지 않는지 테스트한다.
- 전체 테스트와 프로덕션 빌드를 실행한다.
- 로컬 `/ppt-visual-maker`에서 시작·중간·최종 프레임과 자연스러운 소멸을 확인한다.

## 범위 제외

- 이력서 업로드나 실제 사용자 데이터 연동
- 별자리, 점군, 파티클 효과
- 기존 시각화의 기술 목록이나 동작 변경
- 재생·내보내기 UI 변경
