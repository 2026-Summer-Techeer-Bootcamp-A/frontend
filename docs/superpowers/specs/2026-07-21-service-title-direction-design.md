# 방향 발견 서비스 타이틀 시각화 설계

## 목적

기존 `Career Fit 타이틀`과 별개로 제공하는 두 번째 서비스 타이틀 Canvas 시각화다. 여러 후보 경로를 탐색한 뒤 하나의 커리어 방향만 선명하게 남기고, 그 도착점이 `Career Fit` 브랜드로 변하는 과정으로 서비스의 방향 탐색 가치를 전달한다.

## 범위

- 기존 타이틀 시각화와 문제 인식 시각화는 수정하지 않는다.
- 흰 배경의 새 Canvas 시각화 파일과 전용 테스트를 추가한다.
- 고정된 경로와 도착점만 사용하며 사용자 입력과 API를 연결하지 않는다.
- 화면 텍스트는 최종 `Career Fit`과 한 줄 설명으로 제한한다.
- 시각화 목록에는 기존 `Career Fit 타이틀` 바로 다음에 별도 항목으로 등록한다.

## 타임라인

전체 재생 시간은 7,000ms다.

### 0–900ms: 현재 위치

- 화면 중앙 왼쪽에 작은 짙은 점 하나가 나타난다.
- 점 주위에는 한 번만 약하게 퍼지는 원형 파동을 표시한다.
- 타이틀과 설명 문구는 보이지 않는다.

### 700–2,000ms: 후보 경로 전개

- 현재 위치에서 오른쪽으로 향하는 다섯 개의 회색 곡선이 순차적으로 그려진다.
- 경로의 높이, 굴곡, 도착점은 고정 배열로 정의한다.
- 각 경로 끝에는 작은 회색 원형 도착점을 표시한다.
- 별자리, 카드, 기술 칩, 시장 데이터 형태는 사용하지 않는다.

### 1,700–3,300ms: 경로 탐색

- 작은 탐색 빛 하나가 다섯 경로를 위에서 아래 순서로 빠르게 훑는다.
- 탐색 중인 경로만 잠시 진해지고, 나머지는 연한 회색을 유지한다.
- 탐색 빛은 경로 끝에 도착할 때마다 짧게 축소된 뒤 다음 경로 시작점으로 이동한다.

### 3,100–4,500ms: 방향 선택

- 첫째, 둘째, 넷째, 다섯째 경로는 끝부분부터 끊기며 페이드아웃한다.
- 가운데 셋째 경로만 `#18181B`로 선명해지고 시작점부터 도착점까지 다시 한 번 그려진다.
- 선택된 경로 끝의 도착점은 36×36px 기준 크기의 Fit 심볼로 확장된다.

### 4,400–5,300ms: 브랜드 전환

- 선택된 경로가 도착점 방향으로 짧아지면서 Fit 심볼 안으로 흡수된다.
- Fit 심볼은 기존 서비스 타이틀과 같은 짙은 라운드 정사각형과 밝은 대각 형태를 사용한다.
- 심볼이 중앙으로 이동하고 오른쪽에 `Career Fit`이 페이드인한다.

### 5,200–7,000ms: 최종 타이틀

- 브랜드 행 아래에 `데이터로 찾는 나의 커리어 방향`을 한 줄로 표시한다.
- 마지막 1,300ms 동안 심볼, 서비스명, 한 줄 설명을 고정한다.
- 후보 경로, 현재 위치, 탐색 빛은 모두 사라진 상태를 유지한다.

## 시각 스타일

- 배경: `#F7F7F5`
- 선택 경로, 현재 위치, 심볼, 서비스명: `#18181B`
- 후보 경로와 도착점: `#A1A1AA`, `#D4D4D8`
- 설명 문구: `#71717A`
- 색상은 흰색, 검은색, 회색으로 제한한다.
- 글로우는 탐색 빛과 선택 도착점에만 약하게 사용한다.
- 폰트는 기존 Canvas 시각화의 `FONT` 상수를 사용한다.

## 상태 모델

순수 프레임 계산 함수는 진행률과 해상도를 받아 다음 상태를 반환한다.

- 현재 위치의 좌표, 투명도, 파동 진행률
- 다섯 후보 경로의 고정 제어점, 그리기 진행률, 투명도, 선택 여부
- 탐색 빛의 현재 경로 인덱스, 경로 내 진행률, 좌표, 투명도
- 다섯 도착점의 위치, 투명도, 선택 확장 진행률
- 선택 경로의 흡수 진행률
- Fit 심볼의 위치, 크기, 조립 진행률
- 서비스명과 한 줄 설명의 투명도

모든 경로와 타이밍은 고정하며 `Math.random()`을 사용하지 않는다.

## 검증 기준

- 후보 경로는 정확히 다섯 개이며 가운데 셋째 경로만 최종 선택된다.
- 선택 전에는 모든 경로가 회색이고, 선택 후 셋째 경로만 검은색으로 남는다.
- 탐색 빛은 한 시점에 하나의 경로에만 표시된다.
- 최종 프레임에는 경로와 탐색 빛이 사라지고 `Career Fit`과 한 줄 설명만 남는다.
- 같은 진행률과 해상도는 항상 같은 상태를 반환한다.
- 960×540, 1920×1080, 4K에서 경로와 텍스트가 잘리거나 겹치지 않는다.
- 기존 테스트와 프로덕션 빌드가 계속 통과한다.
