# 기술 칩 낙하 · 모노 시각화 설계

## 목적

기존 `기술 칩 낙하`의 순차 낙하와 4단 적층 동작을 유지하면서, 색상 표현을 흑·백·회색만 사용하는 별도 Canvas 시각화로 제공한다. 기존 컬러 버전은 비교와 재사용을 위해 그대로 보존한다.

## 화면 구성

- Career Fit의 어두운 스테이지, 상단 `문제 인식 · 쏟아지는 기술`, 하단 캡션과 바닥선을 유지한다.
- Java, Spring, AWS 등 기존 16개 기술과 8초 재생 시간, 낙하 순서, 반동, 최종 4단 적층 위치를 유지한다.
- 칩은 흰색 5개, 중간 회색 6개, 짙은 회색 5개로 구성하고 각 명도를 적층 전체에 고르게 분산한다.
- 흰색과 중간 회색 칩에는 검은 글자와 어두운 아이콘을, 짙은 회색 칩에는 흰 글자와 밝은 아이콘을 사용한다.
- 짙은 칩이 배경에 묻히지 않도록 밝은 회색 테두리와 얕은 그림자를 적용한다.
- 기술 카테고리별 컬러 의미는 사용하지 않는다. 색상은 오직 무채색 시각 리듬을 위한 장식이다.

## 코드 구조

- 새 파일 `src/ppt/viz/tech-chip-pile-mono.ts`에서 모노 렌더러와 시각화 정의를 제공한다.
- 기존 `tech-chip-pile.ts`가 노출하는 고정 기술 데이터와 `getTechChipPileStates`를 읽어 낙하·적층 계산을 재사용한다.
- 기존 컬러 렌더러와 공통 Canvas 유틸리티는 수정하지 않는다.
- `vizRegistry.ts`에는 새 시각화 import와 항목만 추가한다.
- 시각화 ID는 `tech-chip-pile-mono`, 제목은 `기술 칩 낙하 · 모노`, 카테고리는 `feature`, 재생 시간은 8000ms로 정의한다.

## 데이터 흐름

1. 플레이어가 0~1 범위의 진행률과 캔버스 크기를 렌더러에 전달한다.
2. 모노 렌더러가 기존 상태 계산 함수에서 16개 칩의 위치·회전·투명도·크기를 얻는다.
3. 칩 인덱스를 고정 무채색 팔레트에 매핑한다.
4. 각 칩의 명도에 맞춰 배경, 테두리, 아이콘, 글자색을 그린다.
5. 동일 진행률과 해상도에는 항상 동일한 프레임을 출력한다.

## 오류 및 호환성

- 외부 이미지, 폰트 다운로드, 물리 엔진과 난수에 의존하지 않는다.
- 720p, 1080p, 4K에서도 기존 상태 계산의 비례 좌표를 그대로 사용한다.
- 새 파일이나 레지스트리 연결에 문제가 생겨도 기존 컬러 시각화 파일의 구현은 영향을 받지 않는다.

## 검증

- 모노 팔레트가 흑·백·회색 계열만 포함하는지 테스트한다.
- 16개 기술, 8초, 기능 카테고리, 독립 ID를 테스트한다.
- 기존 상태 계산 함수를 사용해 시작·중간·최종 프레임이 컬러 버전과 같은 낙하·적층 상태인지 테스트한다.
- 전체 테스트와 프로덕션 빌드를 실행한다.
- 로컬 `/ppt-visual-maker`에서 별도 목록 항목과 시작·낙하·최종 적층 프레임을 확인한다.

## 범위 제외

- 기존 컬러 시각화의 색상이나 동작 변경
- 기술 목록, 낙하 궤적, 적층 위치 변경
- 내보내기 UI, 재생 속도, 해상도 기능 변경
