# 모노 기술 더미 질문 장면 설계

## 목적

모노톤 기술 칩이 이미 바닥에 쌓인 상태에서 질문 문구가 페이드인·유지·페이드아웃되는 별도 Canvas 시각화를 만든다. 기술 낙하, 반동, 적층 애니메이션은 포함하지 않는다.

## 화면 구성

- Career Fit의 어두운 `#282828` 스테이지와 Pretendard 타이포그래피를 유지한다.
- 시작 프레임부터 모노톤 기술 칩 16개가 화면 하단에 4단으로 완전히 쌓여 있다.
- 칩은 기존 모노 낙하 시각화와 동일한 흰색 5개, 중간 회색 6개, 짙은 회색 5개의 구성과 최종 적층 위치를 사용한다.
- 칩 더미는 전체 재생 시간 동안 위치, 회전, 크기, 투명도가 변하지 않는다.
- 질문 문구는 칩 더미 위의 비어 있는 중앙 영역에 두 줄로 표시한다.

```text
쏟아지는 기술들,
내 커리어에 필요한 건 무엇일까?
```

- 첫 줄은 중간 회색의 작은 도입 문장, 둘째 줄은 흰색의 큰 핵심 문장으로 표현한다.
- 별, 점군, 입자, 추가 장식과 분야별 컬러는 사용하지 않는다.

## 애니메이션

- 전체 재생 시간은 5000ms다.
- 0.0~0.6초: 문구가 투명도 0에서 1로 페이드인한다.
- 0.6~3.8초: 문구가 완전히 보이는 상태를 유지한다.
- 3.8~4.6초: 문구가 투명도 1에서 0으로 페이드아웃한다.
- 4.6~5.0초: 기술 칩 더미만 남은 정지 상태를 유지한다.
- 페이드는 ease-in-out 보간을 사용한다.

## 코드 구조

- 새 파일 `src/ppt/viz/tech-pile-question.ts`에 고정 칩 데이터, 문구 투명도 계산, Canvas 렌더러, 시각화 정의를 둔다.
- 기존 `tech-chip-pile-mono.ts`에서 `MONO_CHIP_STYLES`를 읽고, `tech-chip-pile.ts`에서 최종 적층 상태를 계산하는 `getTechChipPileStates(1, width, height)`를 읽어 스타일과 위치를 일치시킨다.
- 기존 `tech-chip-pile.ts`, `tech-chip-pile-mono.ts`, `common.ts`는 수정하지 않는다.
- 새 테스트 `tests/techPileQuestion.test.ts`에서 고정 칩 상태, 문구 페이드 시점, 메타데이터를 검증한다.
- `src/ppt/vizRegistry.ts`에는 새 시각화 import와 항목만 추가한다.
- 시각화 ID는 `tech-pile-question`, 제목은 `기술 더미 질문`, 카테고리는 `feature`, 재생 시간은 5000ms로 정의한다.

## 상태 계산

`getTechPileQuestionFrame(progress, width, height)`는 다음 값을 반환한다.

- `chips`: 기존 최종 적층 상태를 복사한 16개 칩
- `questionAlpha`: 현재 진행률에 따른 문구 투명도

동일 입력은 항상 동일한 결과를 반환하며 난수를 사용하지 않는다.

## 검증

- 모든 진행률에서 16개 칩이 보이고 정착 상태인지 테스트한다.
- 시작, 유지, 페이드아웃 종료 시점의 문구 투명도를 테스트한다.
- 시작과 마지막 칩 상태가 동일한지 테스트한다.
- 시각화 ID, 기능 카테고리, 5000ms 재생 시간을 테스트한다.
- 전체 테스트와 프로덕션 빌드를 실행한다.
- 로컬 `/ppt-visual-maker`에서 시작, 문구 유지, 페이드아웃 이후 프레임을 확인한다.

## 범위 제외

- 기술 낙하, 반동, 적층 애니메이션
- 기존 모노 낙하 시각화 변경
- 기술 목록, 칩 스타일, 적층 위치 변경
- 재생·내보내기 UI 변경
