# 채용시장 시장 흐름 레이아웃 수정 설계

## 문제

시장 흐름의 공용 그리드는 한 행을 108px로 고정하지만, `연도별 수요 레이스`의 범프 차트는 232px 고정 높이를 사용한다. 2×2 카드가 확보하는 높이보다 헤더·설명·차트의 실제 높이가 커서 차트가 카드와 다음 캘린더 영역을 침범한다.

## 수정 방향

- 시장 흐름 그리드는 주변 판도 카드에 맞는 기존 108px 행 높이를 유지한다.
- 범프 차트 높이는 232px에서 128px로 줄여 기존 2×2 카드 안에 맞춘다.
- 레이스 카드에만 전용 클래스를 적용해 공용 헤더의 24px 상단 마진을 제거한다.
- 요약 하단 여백을 줄이고 SVG 상단 패딩은 20px로 조정해 제목·필터는 위로, 그래프는 그보다 살짝 아래에 둔다.
- 실데이터의 모든 교차점을 표시하지 않고 차트 전체에서 순위 변화 폭이 가장 큰 교차 하나만 `추월` 배지로 표시한다.
- `추월` 배지는 실제 교차점의 바로 위에 작게 배치해 의미를 유지하면서 선 가림을 최소화한다.
- 순위 이력에 빈 연도가 포함되면 끊긴 선 조각을 그리지 않고 연속된 폴백 데이터를 사용한다.
- 그룹 점유율 API가 세 항목 미만만 반환하면 불완전 응답으로 보고 기존 상위 3개 목록을 유지한다.
- 수요·관계·글로벌 섹션의 기존 108px 행과 강제 row span은 유지한다.
- 데스크톱 4열, 중간 2열, 모바일 1열 구조는 그대로 유지한다.
- 카드 내부 콘텐츠를 잘라 숨기지 않고 범프 차트의 축·라벨 가독성을 보존한다.

## 검증

- CSS 레이아웃 계약 테스트로 시장 흐름 행 높이가 다른 그리드와 분리되어 있는지 확인한다.
- 차트에 `추월` 배지가 하나만 렌더링되고 실제 교차점 기준 좌표를 쓰는지 회귀 테스트로 확인한다.
- 프로젝트 테스트와 프로덕션 빌드를 실행한다.
- 가능하면 채용시장 화면을 데스크톱 폭에서 렌더링해 연도별 레이스와 캘린더가 겹치지 않는지 확인한다.
