# 이력서 더미 → 컬러 기술 칩 전환 시각화 설계

## 목적

PPT Visual Maker에 이력서 6장이 더미로 쌓인 뒤 화면 위로 완전히 퇴장하고, 이어서 공식 로고를 포함한 컬러 기술 칩 30개가 낙하해 바닥에 쌓이는 독립 Canvas 시각화를 추가한다. 이 장면은 이력서 안에 흩어진 기술 정보가 다수의 기술 스택으로 드러나는 문제 인식 흐름을 텍스트 없이 전달한다.

## 변경 범위

- 새 Canvas 렌더 파일 `src/ppt/viz/resume-tech-transition.ts`를 추가한다.
- `src/ppt/vizRegistry.ts`에 새 기능 시각화 항목 하나를 등록한다.
- 전용 회귀 테스트 `tests/resumeTechTransition.test.ts`를 추가하고 기본 테스트 명령에 포함한다.
- 기존 `resume-stack-clean.ts`, `tech-chip-pile.ts`, 모노톤 기술 칩 시각화, 기술 더미 질문 시각화는 수정하거나 삭제하지 않는다.
- 기존 플레이어, 배속, 타임라인 탐색, 해상도 선택, MP4 내보내기 파이프라인을 재사용한다.
- 새 런타임 의존성은 추가하지 않고 현재 설치된 `simple-icons`를 공식 로고 소스로 사용한다.

## 장면 구성

### 1. 이력서 적층

- 개인정보나 실제 문장 없이 프로필 원과 중립적인 텍스트 라인으로 구성된 이력서 6장을 사용한다.
- 여섯 장은 서로 다른 화면 가장자리에서 진입해 중앙의 정돈된 더미로 쌓인다.
- 마지막 장이 안착한 뒤 짧은 정지 시간을 두어 완성된 이력서 더미를 분명하게 보여준다.

### 2. 분리된 장면 전환

- 완성된 이력서 더미 전체가 위로 이동하면서 페이드아웃한다.
- 이력서의 불투명도가 0이 되고 화면 밖으로 퇴장한 이후에만 기술 칩 낙하를 시작한다.
- 이력서와 기술 칩은 어떤 프레임에서도 동시에 보이지 않는다.
- 두 장면 사이에 약 `0.2초`의 빈 스테이지를 두어 전환을 명확하게 한다.

### 3. 기술 칩 낙하와 적층

- 30개 기술 칩이 서로 다른 X 위치와 회전을 가진 채 빠르게 순차 낙하한다.
- 각 칩은 바닥 또는 먼저 놓인 칩 위에 도달하면 짧게 반동한 뒤 고정된 최종 위치에 안착한다.
- 최종 더미는 아래가 넓고 위로 갈수록 좁아지는 다단 구조로 구성하며 모든 기술명이 읽히도록 과도한 겹침을 피한다.
- 마지막 구간에는 모든 칩이 정지한 완성 상태를 유지한다.

## 10초 타임라인

1. `0.0~0.3초`: 빈 다크 스테이지를 유지한다.
2. `0.3~4.0초`: 이력서 6장이 순차 진입해 중앙에 쌓인다.
3. `4.0~4.3초`: 완성된 이력서 더미를 짧게 유지한다.
4. `4.3~5.2초`: 이력서 더미가 위로 이동하며 완전히 페이드아웃한다.
5. `5.2~5.4초`: 아무 요소도 없는 빈 스테이지를 유지한다.
6. `5.4~9.2초`: 컬러 기술 칩 30개가 낙하하고 반동 후 바닥에 쌓인다.
7. `9.2~10.0초`: 모든 기술 칩이 정지한 최종 상태를 유지한다.

| 분야 | 기술 |

| 언어 | Java, Python, JavaScript, TypeScript, Go, Kotlin, Swift, Rust, Ruby, PHP |
| 프레임워크·데이터 | Spring, React, Node.js, Django, FastAPI, Vue, Next.js, MySQL, PostgreSQL, Redis, MongoDB, Kafka, GraphQL |

## 기술 칩 디자인

- 스테이지 배경은 기존 컬러 기술 칩 시각화와 같은 다크 계열을 사용한다.
- 칩은 기술 분야에 따라 블루, 그린, 앰버 계열의 컬러 배경과 테두리를 사용한다.
- 모든 기술명은 흰색으로 통일하고 Pretendard 600을 사용한다.
- 기존 이니셜 원은 제거하고, 왼쪽 아이콘 영역에 `simple-icons`의 공식 SVG 경로를 Canvas `Path2D`로 렌더링한다.
- 로고는 각 기술의 공식 브랜드 컬러를 유지한다. 칩 배경과 명도 차가 부족한 로고에는 작고 중립적인 아이콘 받침을 적용해 식별성을 확보한다.
- 칩 너비는 기술명 길이에 따라 달라지는 pill 형태로 계산한다.
- 그림자와 반동은 기존 Career Fit 시각화의 절제된 깊이감과 모션을 따른다.
- 모든 위치와 크기는 스테이지 크기에 비례해 720p, 1080p, 4K에서 같은 구도를 유지한다.

## 결정적 상태와 렌더링 계약

- 렌더 함수는 기존 `VizRender(ctx, width, height, progress, timestamp)` 계약을 따른다.
- 상태 계산은 Canvas와 분리된 순수 함수 `getResumeTechTransitionState(progress, width, height)`로 제공한다.
- 이력서와 칩의 시작 위치, 시작 시각, 회전, 최종 위치를 고정 데이터로 정의하고 `Math.random()`은 사용하지 않는다.
- 동일한 진행률과 해상도에서는 항상 동일한 상태를 반환해 미리보기, 탐색, 배속 재생, MP4 결과를 일치시킨다.
- 이력서는 `progress < 0.52`에서만 표시하고 기술 칩은 `progress >= 0.54`에서만 표시한다. 따라서 `0.52~0.54` 구간에는 두 요소를 모두 표시하지 않는다.
- Canvas 컨텍스트 상태는 카드와 칩마다 `save()`와 `restore()`로 격리한다.
- 공식 로고 경로를 사용할 수 없는 예외 상황에서는 칩이나 전체 렌더를 중단하지 않고 로고 영역만 비운 채 기술명은 유지한다.

## 검증 기준

- 로컬 `/ppt-visual-maker` 기능 시각화 목록에 새 항목이 별도로 표시된다.
- 기존 이력서 더미, 컬러 기술 칩 낙하, 모노톤 시각화, 기술 더미 질문 항목이 그대로 유지된다.
- 이력서 6장이 먼저 쌓이고 완전히 퇴장한 이후에만 기술 칩 낙하가 시작된다.
- 어느 진행률에서도 이력서와 기술 칩이 동시에 표시되지 않는다.
- 승인된 30개 기술이 각각 한 번씩 표시되고 모든 기술명은 흰색이다.
- 이니셜 대신 공식 로고가 표시되며 로고는 공식 브랜드 컬러를 사용한다.
- 마지막 `0.8초` 동안 기술 칩의 위치와 회전이 완전히 동일하다.
- 720p, 1080p, 4K에서 요소가 스테이지 밖으로 잘리거나 기술명이 읽을 수 없을 정도로 겹치지 않는다.
- 재생, 일시정지, 타임라인 탐색, `0.5~3배속`이 정상 동작한다.
- Chrome에서 1080p MP4 내보내기가 가능하며 미리보기와 같은 프레임 순서를 가진다.
- 전체 테스트와 `npm run build`가 통과한다.
