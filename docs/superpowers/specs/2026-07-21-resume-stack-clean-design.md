# 정돈된 이력서 6장 스택 시각화 설계

## 목적

PPT Visual Maker에 이력서 6장이 화면 밖에서 한 장씩 들어와 중앙에 정돈된 더미로 쌓이는 독립 Canvas 시각화를 추가한다. 이 장면은 문제 인식 서사의 첫 단계로 사용하며, 이번 범위에서는 기술 추출·산포·문제 문구를 포함하지 않는다.

## 변경 범위

- 새 Canvas 렌더 파일 `src/ppt/viz/resume-stack-clean.ts`를 추가한다.
- `src/ppt/vizRegistry.ts`에 새 시각화 항목 하나를 등록한다.
- 기존 `src/ppt/viz/problem.ts`와 기존 디자인 토큰·공용 렌더 헬퍼의 구현은 수정하지 않는다.
- 기존 플레이어, 타임라인, 배속, 해상도 선택, MP4 내보내기 파이프라인을 그대로 사용한다.

## 시각 디자인

- 스테이지 배경은 PPT Visual Maker의 현재 배경색을 사용하며 기본값은 `#282828`이다.
- 이력서 종이는 순백색 대신 쿨 그레이 계열을 사용한다.
- 카드의 모서리, 테두리, 그림자는 기존 Career Fit 디자인의 절제된 깊이감을 따른다.
- 이력서 내부는 개인정보나 실제 문장을 표시하지 않고, 프로필 원과 중립적인 텍스트 라인만 사용한다.
- 카드 6장은 같은 규격을 사용하되 내부 라인 길이와 초기 회전에 작은 차이를 둔다.
- 마지막 프레임은 흐트러짐이나 과부하 표현 없이 중앙의 정돈된 이력서 더미로 멈춘다.

## 6초 타임라인

1. `0.0~0.4초`: 빈 스테이지를 유지한다.
2. `0.4~4.2초`: 이력서 6장이 약 0.6초 간격으로 좌우·상하의 서로 다른 시작점에서 진입한다.
3. 각 카드는 작은 회전을 동반해 들어오며 감속 이징으로 중앙 스택 위치에 안착한다.
4. 먼저 놓인 카드는 다음 카드가 올라올 때 미세하게 아래로 눌려 더미의 깊이를 만든다.
5. `4.2~5.0초`: 마지막 카드가 놓이고 카드 간 간격·회전·그림자가 안정된다.
6. `5.0~6.0초`: 완성된 스택을 움직임 없이 유지해 발표 화면과 MP4 마지막 프레임을 안정적으로 보여준다.

## 렌더링 계약

- 렌더 함수는 기존 `VizRender(ctx, width, height, progress, timestamp)` 계약을 따른다.
- 위치와 상태는 `progress` 값으로만 계산해 탐색과 MP4 내보내기의 동일성을 보장한다.
- 카드별 변형값은 고정 배열 또는 결정적 시드값으로 계산하고 `Math.random()`은 사용하지 않는다.
- 레이아웃은 고정 픽셀 좌표 대신 스테이지 크기에 비례해 계산하여 720p, 1080p, 4K에서 같은 구도를 유지한다.
- `prefers-reduced-motion`, 배속, 타임라인 탐색은 기존 `useVizPlayer` 동작을 그대로 따른다.

## 레지스트리 항목

- ID: `resume-stack-clean`
- 제목: `이력서 더미`
- 부제: `흩어진 이력서 6장이 하나의 더미로 모이다`
- 카테고리: `feature`
- 재생 주기: `6000ms`

## 오류 및 격리

- 새 렌더 파일에 문제가 생겨도 기존 `problem.ts`와 다른 시각화 렌더에는 영향을 주지 않아야 한다.
- Canvas 컨텍스트 상태는 카드별 `save()`/`restore()`로 격리한다.
- 새 시각화 등록 외에는 기존 시각화 목록과 순서를 변경하지 않는다.

## 검증 기준

- 로컬 `/ppt-visual-maker` 좌측 기능 시각화 목록에 새 항목이 표시된다.
- 6장이 순서대로 진입하고 마지막 1초 동안 정돈된 스택이 정지한다.
- 재생, 일시정지, 타임라인 탐색, 0.5~3배속이 정상 동작한다.
- 720p, 1080p, 4K에서 카드가 잘리거나 겹침이 무너지지 않는다.
- Chrome에서 1080p MP4 내보내기가 완료되고 미리보기와 같은 프레임 순서를 가진다.
- 기존 테스트와 `npm run build`가 통과한다.
