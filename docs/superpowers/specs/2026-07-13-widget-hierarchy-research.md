# 위젯 위계·레이아웃 리서치 (채용/대시보드 서비스 벤치마킹)

날짜: 2026-07-13. 목적: 대시보드·시장 페이지의 "위젯 많으나 위계·레이아웃 중구난방" 문제 해결 근거.

## 벤치마킹 핵심
- **채용 플랫폼은 개인화 vs 시장을 페이지로 분리한다.** 사람인(Main 개인화 / Sub 목록·검색), Indeed(홈 개인화 / salaries 별도), 잡코리아(상단 "오늘의 AI Insight" 히어로 + 5개 테마 shelf: 합격가능성높은/유사직무/직무별TOP/최신인기/닮은꼴). 원티드(인사이트·교육 먼저, 목록은 2번째 화면, 전부 카드 shelf). LinkedIn(좌 레일=신원·스탯 / 중앙=피드 / 우 레일=발견).
- **BI 대시보드 표준 레이아웃**: 사이드바 → 상단 KPI 스코어카드 4~6개(큰 값+델타+스파크라인) → KPI를 설명하는 차트 → 하단 상세 테이블. 크기=중요도, 히어로 1개, top-left가 프라임. 관련 위젯은 **라벨 붙은 섹션**으로 그룹(제목 있는 섹션은 3배 빨리 찾음). 한 화면 5~9위젯. 진행적 공개. 존 내 카드 높이 일관. 색은 절제(2 서피스/2 텍스트/1 액센트). "벤치마크 없는 지표는 노이즈."
- Grafana: top-left 프라임, 상단 KPI 개요행, row로 관련 패널 묶기(바인더). Datadog: 관련 위젯 항상 그룹(collapsible+헤더), macro→micro, 커지면 탭 분리. Baymard: 카드 스타일 일관, 크기는 콘텐츠따라 가변 허용(리스트=긴카드+내부스크롤).

## 적용 플레이북

### 대시보드(개인화) — inverted pyramid: 상태→맥락→상세
- 히어로 = **커리어 점수**(top-left, 2~3배 크게, 델타 + "무엇이 바꿨나" 한 줄). "Am I OK?" 앵커.
- Zone1 히어로행(above fold): 좌 7~8col 커리어점수 히어로 + 우 4~5col 레일(지원가능 공고 수/CTA + 미니KPI 2~3: 오늘새매칭·북마크·프로필완성). **above fold는 히어로+3~4개만.**
- Zone2 맥락: 최신공고 타임라인(중앙 피드) + 배울기술 패널(4col 레일, 점수를 설명 → 맥락층).
- Zone3 상세: 북마크·보조리스트 하단, 진행적 공개("전체 보기").
- 규칙: 히어로 1개만, 공고성 카드(지원가능/타임라인/북마크)는 시각적으로 동일 시스템.

### 시장(시장흐름) — 3개 라벨 섹션(macro→micro)
- **수요**: 히어로 차트=스킬 수요 리더보드(2/3, 8col) + 우 1/3 레일=응답률. 페이지 히어로존, top-left.
- **트렌드**: 채용추이(시계열) + Hype vs Hire(사분면) 나란히 같은높이.
- **기업·구조**: 산업지문 + 공동출현 네트워크(무거운 탐색층) 마지막, 진행적 공개.
- 규칙: featured 히어로 차트 1개(수요 리더보드)만 크게+한줄 takeaway, 나머지는 한 티어 아래. 각 섹션 헤더 + "무엇을 답하나" 한 줄. 섹션 내 카드 높이 일관. 네트워크/지문은 above fold 금지, 하단.

### 3줄 수용 테스트
1. 5초 글랜스: 페이지의 ONE 메시지(히어로 점수/최다수요 기술)를 말할 수 있나.
2. 실눈/블러: 텍스트 흐려도 스켈레톤만으로 히어로·섹션 경계가 보이나.
3. 섹션명: 모든 그룹에 비즈니스 기능 라벨. 라벨 없는 카드 뭉치 없음.

### 3대 고레버리지
(1) 페이지당 히어로 1개 선정·확대, (2) 평평한 그리드를 32px 간격의 3개 라벨 섹션으로(섹션 내 카드 높이 일관), (3) 무겁고 탐색적인 위젯은 진행적 공개로 하단.
