# 기술 칩 낙하·적층 시각화 설계

## 목적

PPT Visual Maker에 여러 기술 칩이 화면 위에서 순차적으로 낙하하고 바닥에 쌓여 하나의 안정된 더미를 만드는 독립 Canvas 시각화를 추가한다. Career Fit 디자인 시스템을 기준으로 색상·타이포그래피·모서리·그림자·모션을 통일하며 기존 시각화 구현은 수정하지 않는다.

## 변경 범위

- 새 Canvas 렌더 파일 `src/ppt/viz/tech-chip-pile.ts`를 추가한다.
- `src/ppt/vizRegistry.ts`에 새 기능 시각화 항목 하나를 등록한다.
- 전용 회귀 테스트 `tests/techChipPile.test.ts`를 추가하고 기본 테스트 명령에 포함한다.
- 기존 `problem.ts`, `resume-stack-clean.ts`, 공용 디자인 헬퍼, 플레이어, 내보내기 구현은 수정하지 않는다.
- 기존 배속·타임라인·720p·1080p·4K·MP4 내보내기 파이프라인을 재사용한다.

## 기술 데이터와 색상 체계

총 16개 기술 칩을 세 카테고리로 나눈다.

| 카테고리 | 색상 | 기술 |
|---|---|---|
| 언어 | 블루 `#6EA8FE` | Java, Python, TypeScript, Go |
| 프레임워크·데이터 | 그린 `#34D17F` | Spring, React, Kafka, Redis, MySQL, PostgreSQL |
| 클라우드·인프라 | 앰버 `#E2933F` | AWS, Docker, Kubernetes, Git, Terraform, Linux |

색은 칩 전체를 밝게 채우지 않고 다음 요소에만 절제해서 사용한다.

- 반투명 카테고리 배경
- 1px 카테고리 테두리
- 왼쪽 원형 이니셜 아이콘
- 밝은 쿨 그레이 기술명

## 칩 디자인

- 기본 배경: `rgba(255,255,255,0.05)`
- 텍스트: `#F4F4F5`
- 보조 테두리: 카테고리 색상에 투명도를 적용한다.
- 형태: 기술명 길이에 따라 너비가 달라지는 pill.
- 타이포그래피: Pretendard 600.
- 그림자: 낮은 높이의 부드러운 검정 그림자.
- 아이콘: 카테고리 색상 원 안에 기술명의 첫 글자 또는 고정 이니셜을 표시한다.
- 모든 치수는 스테이지 크기에 비례해 720p, 1080p, 4K에서 같은 구도를 유지한다.

## 8초 타임라인

1. `0.0~0.4초`: 빈 스테이지와 바닥 기준선만 표시한다.
2. `0.4~5.8초`: 16개 기술 칩이 약 0.3초 간격으로 서로 다른 X 위치에서 낙하한다.
3. 각 칩은 중력 가속을 표현하는 ease-in 곡선과 미세한 좌우 이동·회전을 갖는다.
4. 바닥 또는 먼저 놓인 칩 높이에 도달하면 짧게 한 번 튕기고 고정된 최종 위치로 안착한다.
5. `5.8~7.0초`: 늦게 도착한 칩이 안정되며 전체 더미의 움직임이 잦아든다.
6. `7.0~8.0초`: 모든 칩이 정지한 완성 상태를 유지한다.

## 적층 구조

- 외부 물리엔진을 추가하지 않는다.
- 칩마다 시작 X, 시작 시각, 초기 회전, 최종 X·Y·회전을 고정 데이터로 정의한다.
- 최종 더미는 바닥 1열을 가장 넓게 두고 위로 갈수록 좁아지는 4단 구조로 만든다.
- 서로 겹치는 칩은 라벨이 읽히는 범위에서만 부분적으로 겹친다.
- 낙하 순서는 아래쪽 칩이 먼저, 위쪽 칩이 나중에 도착하도록 정한다.
- 동일한 `progress`, 해상도에서는 항상 동일한 위치를 반환해야 하며 `Math.random()`을 사용하지 않는다.

## 렌더링 계약

- 렌더 함수는 `VizRender(ctx, width, height, progress, timestamp)` 계약을 따른다.
- 상태 계산은 Canvas와 분리된 순수 함수 `getTechChipPileStates(progress, width, height)`로 제공한다.
- Canvas 컨텍스트 상태는 칩마다 `save()`와 `restore()`로 격리한다.
- 시작·낙하·반동·정지 구간은 `progress`로 계산해 타임라인 탐색과 MP4 렌더가 일치하게 한다.
- MP4 내보내기 중 숨겨지는 표준 상단 라벨·하단 캡션 이외의 화면 문구는 추가하지 않는다.

## 검증 기준

- 로컬 `/ppt-visual-maker` 기능 시각화 목록에 `기술 칩 낙하`가 별도 항목으로 표시된다.
- 16개 칩이 위에서 순차적으로 떨어지고 한 번 반동한 뒤 4단 더미로 안착한다.
- 언어·프레임워크/데이터·클라우드/인프라가 각각 블루·그린·앰버로 일관되게 표시된다.
- 마지막 1초의 칩 상태가 완전히 동일하다.
- 720p, 1080p, 4K에서 칩이 스테이지 밖으로 잘리거나 라벨을 가리는 과도한 겹침이 없다.
- 재생·일시정지·탐색·0.5~3배속이 정상 동작한다.
- Chrome에서 1080p MP4 내보내기가 가능하다.
- 기존 테스트와 `npm run build`가 통과한다.
