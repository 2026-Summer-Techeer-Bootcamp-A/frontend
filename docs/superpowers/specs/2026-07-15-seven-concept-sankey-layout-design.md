# 7개 핵심 개념 Sankey 배치 설계

## 목표

기존 5개 핵심 개념 Sankey에 `실시간·스트리밍`, `클라우드 네이티브`를 추가하고, 7개 개념이 차트 세로 중앙에서 충분한 간격을 두고 읽히도록 한다.

## 개념 구성과 순서

1. MSA·분산
2. 대규모 트래픽
3. 실시간·스트리밍
4. 보안·컴플라이언스
5. 데이터 파이프라인
6. 클라우드 네이티브
7. DevOps·자동화

각 개념은 현재 API 공동출현 데이터의 상위 기술 4개를 사용한다. API 응답에서 개념이 누락되면 기존 `conceptReal.json` 실측 데이터의 상위 기술 4개를 사용한다.

## 차트 배치

- 차트 높이: 500px
- Sankey 위·아래 여백: 각각 30px
- 노드 간격: 24px
- 왼쪽·오른쪽 여백은 기존 25%·18%를 유지해 긴 한글 개념명과 기술명이 잘리지 않게 한다.
- `layoutIterations: 0`과 고정된 개념 배열 순서로 왼쪽 노드 순서를 유지한다.
- 개념 색상은 7개로 확장해 같은 화면에서 색상이 반복되지 않게 한다.

## 테스트와 검증

- 큐레이션 테스트에서 7개 개념과 고정 순서를 검증한다.
- 새 개념이 API 응답에 없을 때 실측 폴백 링크로 보충되는지 검증한다.
- 전체 프론트엔드 테스트와 프로덕션 빌드를 실행한다.
- 데스크톱 시장 화면에서 라벨 잘림, 노드 겹침, 세로 중앙 배치를 육안 확인한다.

## 범위 제외

- Sankey를 다른 차트 유형으로 교체하지 않는다.
- 개념별 상위 기술 개수 4개는 변경하지 않는다.
- 백엔드 API 또는 데이터 집계 방식은 수정하지 않는다.
