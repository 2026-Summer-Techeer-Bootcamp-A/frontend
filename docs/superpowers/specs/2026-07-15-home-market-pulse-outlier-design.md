# 홈 채용 시장 위젯 특이값 처리 설계

## 목표

홈 왼쪽의 `채용 시장` 위젯을 교체하지 않고 유지한다. 잘못 수집된 날짜가 특정 하루에 몰리더라도 이를 정상적인 최근 채용 추세나 합계로 오해시키지 않도록 표시를 바로잡는다. 기존 기술 수요 Top 5, 국내·해외 배지, 이동 동작과 카드 외형은 유지한다.

## 확인된 원인

- 프론트는 `/stats/posting-timeline?days=7` 응답을 검증 없이 모두 합산하고, 최댓값을 스파크라인 축으로 사용한다.
- 백엔드는 오늘 날짜가 아니라 해당 풀의 `MAX(post_date)`를 `as_of`로 사용한다.
- 2026-07-15에 확인한 운영 국내 응답은 2026-07-10 하루가 2,717건이고 주변 날짜는 36~244건이었다.
- 같은 운영 데이터의 최근 365일 비영(0이 아닌) 일자 분포는 중앙값 55건, 상위 5% 값 243건, 최댓값 2,717건이었다.
- 따라서 2,717건은 일반적인 일별 변동이 아니라 수집 배치가 한 날짜에 몰린 특이값으로 판단할 수 있다.
- 시장 페이지의 1년 캘린더도 동일 문제를 인지하고 상위 5% 값으로 시각화 강도만 제한한다.

프론트는 집계된 날짜별 건수만 받으므로 각 공고의 실제 등록일을 복원할 수 없다. 이번 수정은 원본 날짜를 고치는 작업이 아니라, 오염된 집계를 정상 추세처럼 표현하지 않도록 하는 작업이다.

## 데이터 처리

1. 기존 `postingTimeline` API를 `days=365`로 호출한다. 새 엔드포인트는 추가하지 않는다.
   - `국내`와 `해외`는 해당 풀을 한 번 호출한다.
   - `전체`는 국내·해외를 각각 호출한 뒤 같은 날짜의 `total`과 `matched`를 더한다.
2. 응답의 마지막 7개 일자만 스파크라인에 사용한다.
3. 365일 중 건수가 0보다 큰 값만 오름차순으로 정렬한다.
4. 일반적인 시각화 상한은 정렬된 값의 상위 5% 위치 값(P95)으로 정한다.
5. 하루 건수가 `P95 × 2`를 초과하면 특이값으로 분류한다.
6. 스파크라인 좌표를 계산할 때 특이값은 P95로 제한한다. 원래 건수는 변경하거나 버리지 않는다.
7. 최근 7일의 단순 합계는 제거한다. 최근 7일에서 0건과 특이값을 제외한 중앙값을 사용해 `평소 하루 N건`을 표시하고, 특이값이 있으면 `특이값 N일`을 함께 표시한다. 특이값 판정용 P95는 계속 365일 분포를 사용한다.
8. API의 `as_of`를 `기준 YYYY-MM-DD`로 표시한다.

중앙값과 P95는 데이터가 비어 있거나 모두 0인 경우 0으로 처리한다. 365일보다 짧은 응답도 같은 규칙으로 계산하되, 빈 응답은 추이 영역을 표시하지 않는다.

## UI

- 카드 제목 `채용 시장`과 풀 배지는 그대로 둔다.
- `전체` 선택 시 풀 배지는 `전체`로 표시하고, 국내·해외 합산 결과를 사용한다.
- 기존 `최근 7일 새 공고 N건` 문구를 `최근 7일 등록 추이`로 바꾼다.
- 보조 문구는 `평소 하루 N건`으로 표시한다.
- 최근 7일에 특이값이 있으면 같은 줄에 앰버 계열의 `특이값 N일` 텍스트를 표시한다.
- 특이값이 있는 점은 스파크라인에서 일반 선과 구별되는 작은 마커로 표시한다.
- 그래프 높이는 P95 상한을 사용하지만, 마커의 접근성 라벨에는 실제 날짜와 원 건수를 포함한다.
- 추이 영역 아래에 `기준 YYYY-MM-DD`를 표시한다.
- `지금 많이 찾는 기술` Top 5와 각 기술의 공고 검색 이동, `채용 시장 전체 보기`는 변경하지 않는다.
- 현재 카드의 여백, 색상 토큰, 타이포그래피를 재사용하며 새로운 카드나 별도 디자인을 만들지 않는다.

## 상태와 오류 처리

- 로딩 중에는 현재 스켈레톤을 유지한다.
- 기술 점유율 또는 타임라인 요청 중 하나라도 실패하면 현재 정책대로 카드를 숨긴다.
- 타임라인이 비어 있고 기술 점유율만 있으면 추이 영역만 생략하고 기술 Top 5를 표시한다.
- `all` 선택 시 타임라인은 국내·해외 날짜별 합산값을 사용한다. 기술 점유율은 API 허용 상한인 각 풀 Top 100을 받아 canonical 기술명별 `posting_count`를 더하고, 합산 `sample_size`로 `share`를 다시 계산한 뒤 Top 5를 표시한다.
- 전체 풀 요청 네 개 중 하나라도 실패하면 현재 정책대로 카드를 숨긴다.

## 코드 구조

- 날짜 분포 계산은 컴포넌트 JSX와 분리한 순수 함수로 구현한다.
- 순수 함수 입력은 `daily[{date,total}]`, 출력은 최근 7일, 중앙값, P95 상한, 특이값 수를 포함한다.
- `HomeMarketPulse`는 API 상태 관리와 렌더링만 담당한다.
- 기존 시장 캘린더의 특이값 정의와 표현 원칙을 따르되 코드를 직접 결합하지 않는다. 홈 위젯은 7일 스파크라인에 필요한 최소 결과만 사용한다.

## 검증

- 단위 테스트: 최근 365일의 일반 분포 뒤에 운영 응답과 같은 `[65,244,36,65,38,2717,1]` 구간을 둔 fixture에서 2,717건이 특이값으로 판정되고 시각화 값만 제한되는지 확인한다.
- 단위 테스트: 정상 분포에서는 특이값이 만들어지지 않는지 확인한다.
- 단위 테스트: 빈 데이터와 모두 0인 데이터가 안전하게 처리되는지 확인한다.
- 단위 테스트: 국내·해외 타임라인이 날짜별로 합산되고, 기술 점유율이 canonical 기준으로 합쳐져 Top 5 순서와 share가 다시 계산되는지 확인한다.
- 빌드: TypeScript와 Vite 프로덕션 빌드가 통과해야 한다.
- 브라우저: 국내·해외 전환, 스켈레톤, 기준일, 특이값 표시, 기술 필터 이동을 직접 확인한다.

## 범위 밖

- 백엔드 또는 수집기에서 잘못된 `post_date`를 복원하거나 정정하는 작업
- 새로운 통계 API 추가
- 카드 교체 또는 홈 왼쪽 컬럼 재디자인
- 기술 Top 5와 대시보드 위젯 구성 변경
