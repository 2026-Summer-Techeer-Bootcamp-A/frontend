# TESTER.md — 커리어 포지셔닝 대시보드 평가 지침 (AI 에이전트용)

> 이 문서는 **Claude · Gemini · GPT 세 AI에게 동일한 기준으로** 이 데모 프론트엔드를 평가시키기 위한 지침서다.
> 각 AI는 아래 **3개 Role**을 모두 수행하고, **§7 산출 형식**대로 점수·근거·개선점을 반환한다.
> 평가자는 "잘 만들었다"는 두루뭉술한 칭찬이 아니라 **구체적 근거(화면·수치·상호작용)**를 들어 채점한다.

---

## 0. 한 줄 정의 & 평가의 본질

**정의**: 이력서(보유 기술)를 기준으로, 실수집 채용공고 **124,237건** 위에서 *내 시장 위치·커버리지·기회·기업*을 계산해 보여주는 모바일 커리어 대시보드.

**평가의 본질(가장 중요)** — 이 프로덕트가 아래 넷을 실제로 해내는지 냉정하게 검증하라:
1. **Wow-Point**가 있는가 (첫 5초에 "오"가 나오는가).
2. **메인/대시보드가 핵심을 말하는가** (히어로가 잡동사니가 아니라 "단 하나의 중요한 것"을 말하는가).
3. **단순 정보 나열/차트가 아니라 개인화**인가 (모든 지표가 "내 기술" 기준으로 재계산되는가, 아니면 그냥 시장 통계를 나열한 대시보드인가).
4. **여러 지표를 조합해 "맞추고 최적화된 정보"**를 산출하는가 (커버리지·갭·수요·매칭을 엮어 *행동 가능한 결론*을 주는가, 아니면 예쁜 차트 나열인가).

> ⚠️ 채점 편향 주의: 화면이 많고 예쁘다는 이유로 후하게 주지 말 것. "차트가 많다 ≠ 좋다." 핵심은 **개인화 + 멀티지표 최적화 + 정직성**이다.

## 1. 평가 전 준비 (반드시 실제로 조작)

- 실행: `npm i && npm run dev` → 표시되는 `localhost:PORT`.
- **우상단 "데모 페이지 · 이동"** 리모컨으로 전 화면 순회 가능.
- 반드시 **직접 클릭·스크롤·스와이프**하며 상호작용을 확인하라(정적 스샷 판단 금지).

## 2. 세 개의 평가 Role

각 Role은 **다른 렌즈**로 본다. 세 결과의 합의/충돌이 진짜 품질 신호다.

### Role A — 사용자 (첫인상 · 체감 개인화 · 쓰고 싶은가)
- **던질 질문**: "처음 켰을 때 무엇이 먼저 보이나? 그게 '나에 대한 것'으로 느껴지나? 5분 뒤 다시 열 이유가 있나? 어지럽지 않고 다음 행동이 분명한가?"
- **본다**: 첫 화면의 명료함, 히어로 숫자의 의미 전달, 개인화 체감("내 기술이라서 이렇게 나온다"가 보이는지), 인터랙션의 즐거움(스와이프·게이지·시트), 산만함(정보 과부하) 여부.
- **감점**: 홈이 위젯 창고, 무슨 숫자인지 모름, 두 번 눌러야 정보가 나옴, 개인화 근거가 안 보임.

### Role B — 전문가 (프로덕트/디자인/데이터)
- **던질 질문**: "정보 위계가 개요→드릴다운으로 서 있나? 데이터가 정직한가(합산 금지·기준일·모수)? 차트가 값=진하기·tabular-nums 등 원칙을 지키나? 디자인 시스템 Do/Don't를 위반하지 않나? AI 클리셰(보라/핑크/Sparkles/무지개)가 없나?"
- **본다**: §5 화면별 체크 + §6 감점 함정. 특히 **정직 표기**와 **DS 일관성**.
- **감점**: 근거 없는 수치, "합격 확률"류 표현, 연봉 위젯, 좌측 스트라이프 만능처방, 무지개 차트, 응답률 같은 없는 데이터의 날조.

### Role C — 면접관 (기술 깊이 · 차별성 · 방어가능성)
- **던질 질문**: "이게 GPT로 하루 만에 찍어낸 대시보드 템플릿과 뭐가 다른가? 여러 지표를 실제로 **조합·최적화**하나, 아니면 각 차트가 독립적인가? 하나의 인터랙션(예: What-if)이 *실데이터로 재계산*되나, 하드코딩된 연출인가? 확장/방어가 되나?"
- **본다**: 개인화의 진위(내 스킬 바꾸면 결과가 바뀌나), 멀티지표 매칭(공고 매칭·커버리지·기회가 한 축으로 엮이나), 실데이터 근거(요즘의 시장·트렌드가 진짜 수치인가), 구현 난이도(지도 클러스터·컨텍스트·이력서별 스킬 등).
- **감점**: 개인화가 껍데기(스킬 바꿔도 불변), 지표들이 서로 무관, "데모용 가짜"가 핵심 주장을 무너뜨림.

## 3. 공통 평가 축 & 가중치 (100점)

| # | 축 | 가중치 | 판정 기준(요지) |
|---|---|---|---|
| 1 | **Wow-Point** | 15 | 첫 5초에 강한 인상. 흔한 템플릿과 즉시 구별되는 한 방(예: 히어로 점수+What-if 히스토그램, 지도 컨텍스트, 사분면). |
| 2 | **메인/대시보드 핵심** | 15 | 히어로가 "단 하나(커버리지 점수)"를 말하고 나머지는 그 아래 위계로 정렬. 3막 서사(점수→분포/배우면→시장). |
| 3 | **개인화 (내 기술 기준)** | 20 | 모든 지표가 이력서 기술로 재계산. "보유/미보유", 갭, 매칭%가 *나* 기준. 시장 통계 단순 나열이면 대폭 감점. |
| 4 | **멀티지표 매칭·최적화** | 20 | 커버리지·갭·수요·공고매칭·연차·기업을 **엮어** 행동 결론 산출(예: What-if로 "이거 배우면 도달 공고 +N", 사분면으로 "공략 지점", 규모/매칭 정렬). 독립 차트 나열이면 감점. |
| 5 | **데이터 정직성** | 12 | 기준일+모수(N) 병기, 글로벌/국내 합산 금지·토글, 합격확률 금지→커버리지, 연봉 금지, 없는 필드(응답률) 미표시, himalayas 시계열 제외. |
| 6 | **디자인 시스템 일관성** | 10 | 단일 액센트(슬레이트블루), 사각 배지, tabular-nums, 절제된 라운드/섀도, AI 과장식 없음. Do/Don't 준수. |
| 7 | **인터랙션 디테일** | 8 | iOS식 전환, 스와이프/방향키, 시트 스프링, 지도 클러스터/컨텍스트, Toss식 디테일(블러·비네팅·스크롤 반응). |

> **개인화(3) + 멀티지표(4) = 40점**이 이 프로덕트의 정체성이다. 여기서 무너지면 총점 상한을 60으로 캡한다.

### Role C · 면접관 (30점)
| 항목 | 1점 | 5점 |
|---|---|---|
| 개인화 진위 | 스킬 바꿔도 불변(껍데기) | 이력서/스킬 변경→결과 재계산 |
| 멀티지표 결합 | 차트 독립 | 커버리지·갭·수요·매칭이 한 축으로 |

> 15점(Role A) + 30 + 30 = **75점**을 Role 합산으로, §3 가중 총점과 **교차검증**한다(§8).

## 5. 화면별 반드시 확인할 포인트

**홈 `/`**
- 히어로 = **시장 커버리지 단일 숫자 + "상위 N/20"** (합격확률 아님).
- 커버리지 **분포 히스토그램**(50% 문턱, 도달=파랑) + **What-if 칩**: 칩 탭 시 분포/도달 공고 수가 **실시간 재계산**되는가? (하드코딩이면 감점)
- "요즘의 시장" = **국내 실데이터 근거**("국내 공고 2022 99건 → 2026 478건" 류)가 붙는가? "기회 기술/내 강점" 같은 편집성 문구 대신 담백한 사실인가?
- 맞춤 공고: 회사 로고·**규모 tier(대/중견/중소)**·매칭%·D-day. 카드 탭 → 바로 상세.

**시장 `/market`**
- **내 기술 vs 시장 사분면**: 보유(파랑)/미보유·고수요(주황) 구분, 버블=공고수. "공략 지점"이 공간적으로 읽히나?
- 수요 막대(보유 강조) → 탭 → 드릴다운(동반기술·추이·산업 히트맵) **디스클로저**.

**지도 `/map`**
- **실지도(Leaflet)** + **지역 클러스터**(공고수 + 바깥 링=그 지역 **평균 매칭률**).
- 겹침 없이 클러스터링되나? **충분히 확대 후** 클러스터 클릭 시 그 지점에서 **컨텍스트 메뉴**(회사 목록)가 부드럽게 나오나?
- 회사 마커/메뉴 항목 → **리치 시트**(로고·매칭·tier·기술칩·상세보기).
- 상단 "화면 내 보이는 회사 3"이 지도 이동 시 스켈레톤과 함께 갱신되나?

**마이 `/resume`**
- 이력서 **1/2/3 전환 시 스킬셋·커버리지가 실제로 달라지나?** (데이터 지향=Python/SQL/Pandas/Airflow 61% 등)
- 보유 기술 **토큰 칩 삭제 + 추가(전체 기술 검색·실제 브랜드 아이콘)** 편집이 되나? 편집이 **한 곳(마이)**에만 있나?

**공고 `/jobs`** — 검색·국내/글로벌·전체/컴팩트·정렬(매칭/규모/최신)·최상단 버튼.

**디자인 시스템 `/design-system`** — Do&Don't(AI 클리셰 반면교사), 위젯 킷, **Toss 디테일 10**(블러·비네팅·스크롤 반응 헤더 등) 실제 동작.

## 6. 감점 함정 (있으면 즉시 지적)

**DS Don't 위반**: 다색 액센트, pill 남용, 무지개 차트, **좌측 액센트 스트라이프 만능처방**, 그림자 0 또는 shadow-2xl, 동심원 라운드 깨짐.
**AI 클리셰**: AI 영역에 보라/핑크/**Sparkles 아이콘**, 이모지 남발, 과장 문구("최고의", "기회 기술" 같은 편집성 카피).
**데이터 부정직**: 근거 없는 수치, "합격 확률", **연봉 위젯**, 글로벌+국내 **합산**, 기준일/모수 누락, 없는 필드(응답률) 날조.
**개인화 껍데기**: 이력서/스킬을 바꿔도 지표 불변, What-if가 연출.
**산만/중복**: 같은 기능이 두 곳, 2단계로 도달, 텍스트 군더더기.

## 7. 각 AI가 반환할 산출 형식

각 AI(Claude/Gemini/GPT)는 아래를 **그대로** 채워 반환한다.

```json
{
  "evaluator": "claude|gemini|gpt",
  "weighted_score": { "wow": 0, "hero": 0, "personalization": 0, "multi_metric": 0, "honesty": 0, "ds_consistency": 0, "interaction": 0, "total_100": 0 },
  "roles": { "user_25": 0, "expert_30": 0, "interviewer_30": 0 },
  "capped": false,               // 개인화+멀티지표 < 24면 true, total_100 상한 60
  "wow_points": ["구체적 한 방 3개"],
  "personalization_evidence": ["스킬/이력서 변경→결과 변화를 직접 확인한 사례"],
  "multi_metric_evidence": ["여러 지표가 하나의 결론으로 엮인 사례(What-if/사분면/정렬 등)"],
  "honesty_findings": ["정직/부정직 근거"],
  "ds_violations": ["발견한 Don't 위반, 없으면 빈 배열"],
  "top_strengths": ["3개"],
  "top_gaps": ["개선 우선순위 3개, 각 화면·근거 포함"],
  "verdict": "한 문단: 단순 정보나열/차트 대시보드인가, 아니면 개인화·멀티지표 최적화 프로덕트인가에 대한 최종 판정"
}
```

## 8. 최종 종합 산식

1. §3 **가중 총점(100)** 계산.
2. **개인화(20)+멀티지표(20) 합이 24점 미만이면** → `capped=true`, 총점 상한 **60**으로 강등(이 프로덕트의 정체성 실패).
3. §4 **Role 합(75)**을 100 환산해 가중 총점과 **±10점 이내 정합**하는지 확인. 크게 벌어지면 근거를 재검토.
4. 세 AI의 총점 **중앙값**을 최종점으로, 세 `verdict`의 공통·상충 지점을 요약.
