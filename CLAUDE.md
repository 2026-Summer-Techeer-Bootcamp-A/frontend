# frontend 저장소 규칙

## 커밋 메시지와 PR 문체 규칙 (필수)

커밋 메시지 본문과 PR 본문은 ../tech-breakthrough/write_rule.md의 문체 기조를 따르되, 산문 문단이 아니라 리스트로 딱딱 정리한다. 제목 줄은 기존 컨벤션 type(scope): 요약 형식을 그대로 쓴다.

- 본문은 불릿 리스트로 쓴다. 항목마다 무엇을 왜 바꿨는지가 드러나게 쓰고, 결과가 달라진 점이 있으면 같은 항목 안에서 이어서 적는다.
- AI가 자주 쓰는 기호를 쓰지 않는다. 가운뎃점, 줄표, 화살표, 이모지, 별표 강조를 금지한다.
- backtick을 쓰지 않는다. 코드 식별자와 파일명도 일반 텍스트로 적는다.
- 종결은 평서체(다, 이다, 했다, 였다)로 담백하게 둔다. 상투적 표현과 과장된 미사여구를 배제한다.
- 괄호 안에 괄호를 넣지 않는다.
